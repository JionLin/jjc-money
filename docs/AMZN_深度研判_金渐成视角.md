## 第二步：Logic Mapping（逻辑映射）

基于 22-26year 文件夹的系统检索，提取金渐成关于 AMZN 的三大核心逻辑：

### 2.1 "不看 PE 看现金流" — 估值哲学

> *"去持有一些业绩增长更快、盈利能力更高、研发能力更强，能推动科技和效率大幅进步的公司。"*
> — 2024-05-24《拒绝新闻的生活》

**核心逻辑**：
- 金渐成对 AMZN 的估值从不看 P/E 绝对值，而看**经营性现金流的持续增长能力**
- 关键判据：OCF 能否持续"滚出"高质量资产（AWS 产能、物流网络、AI 基建）
- **当前评估**：$114B OCF ✅ → 但 $75B CapEx 意味着处于"重投入期"，FCF 暂时受限

### 2.2 "AWS 利润中心" — 增长引擎逻辑

> *"亚马逊和Costco的模式，都属于很有意思的可循环模式。可以长期拿着，属于典型的长牛股。"*
> — 2026-01 月文

**作者历史判断**：
- AWS 被视为 AMZN 的**真正价值锚点**，零售只是"流量入口+规模护城河"
- **隐忧已现**（2025-08 月文）：作者对 AWS 增速落后 Azure/GCP 表达了明确担忧
  - AWS 17% vs Azure 33% vs GCP 28% → **增速差距在扩大**
  - 如果 AWS 市场份额持续流失，"利润中心"叙事将遭到挑战
- **但长期未动摇**：作者仍将 AMZN 定位为"负成本长期持仓"的标的

### 2.3 "零售效率极限" — 机器人公司逻辑

> *"亚马逊本质上是一家大型机器人公司和AI受益者，未来的降本核心在自动配送。"*
> — 2026-01 月文

**核心洞见**：
- 金渐成不将 AMZN 视为"电商公司"，而是**物流+AI+云计算**的三位一体
- 区域化已完成第一轮降本，下一轮降本取决于**自动驾驶配送**（远期催化剂）
- 当前零售效率提升已进入**边际递减阶段**，增量需依赖技术突破

### 2.4 "2-3-3-2 法" — 仓位管理模型

> *"我在二级市场上投资，一般都比较谨慎，看准了比较有把握才下手。"*
> *"里面的每一个数字，都是可以拆分成更小的数字，挺灵活的。"*
> — 2024-05-20《一般从来不一般》

**建仓逻辑**：
| 阶段 | 比例 | 触发条件 |
|------|------|----------|
| 试探仓 | 20% | 基本面确认+价格进入目标区间 |
| 确认仓 | 30% | 上行趋势凸显 |
| 主力仓 | 30% | 大阳线/拉升势头明确 |
| 收尾仓 | 20% | 调整期结束+上行继续 |

**减仓逻辑**：
| 阶段 | 比例 | 策略 |
|------|------|------|
| 首轮减持 | 20%（拆为 5%×4） | 灵活调整，市场疲软则加大 |
| 第二轮 | 30%（分批） | 接近预期目标 |
| 保留持仓 | 50% | 负成本，长期持有 |

> *"投资的首要前提是保值/保本，其次才是增值/收益。"*

**减仓阈值**：如果距预期目标已达 80%，最后 20% 建仓不执行。

## 第三步：Synthesis（综合研判）

### 3.1 "越涨越便宜"检验

> *"PEG越低，公司就越便宜。股价明明在涨，估值却越涨越便宜，这就是成长的力量。"*
> — 2024-05-24《拒绝新闻的生活》（以NVDA为例）

**AMZN 当前 PEG 测算**：

```
Forward P/E ≈ 31x
2026 EPS Growth ≈ 11%
PEG = 31 / 11 ≈ 2.8

对比参考：
- NVDA PEG (2024年时) ≈ 0.3–0.5 → 典型"越涨越便宜"
- AMZN PEG = 2.8 → ❌ 不符合"越涨越便宜"条件
```

> [!CAUTION]
> **结论：AMZN 当前不满足"越涨越便宜"条件。**
> PEG = 2.8 远高于 1.0 的临界值。这意味着股价涨幅已超过盈利增速，估值处于**"合理偏贵"区间**，而非金渐成定义的"成长压缩估值"状态。

### 3.3 2-3-3-2 法当前适用性评估

#### 情景 A：已有仓位（负成本状态）

如果你已在 $190–$225 区间建仓并通过部分卖出实现负成本：

```
✅ 当前策略：持有，不加仓
✅ 减持计划：按 2-3-3-2 法反向操作
   - 如果涨至 $280+，考虑减持首个 5%
   - $300+ 加速减持节奏
   - 保留 50% 作为"负成本永久仓"
```

#### 情景 B：空仓想建仓

```
⚠️ 当前不建议建仓
   - 价格远离作者锚点（高出 15-35%）
   - PEG = 2.8，不满足"便宜"条件
   - Q1 2026 财报（4/29）是关键变量

📌 等待的触发点：
   1. 财报后回调至 $225 以下 → 启动 20% 试探仓
   2. $215 以下 → 启动 30% 确认仓
   3. $200 以下 → 启动 30% 主力仓（"捡血筹码"区间）
   4. $190 以下 → 完成最后 20%
```

#### 情景 C：轻仓持有

```
🟡 持有观察，不追高
   - 等 Q1 2026 财报（4/29）确认 AWS 利润率走向
   - 如 AWS 利润率 >38% + 增速回升 → 逻辑强化，可适度加仓
   - 如 AWS 利润率 <35% + 增速继续落后 → 考虑减仓等回调
```
