## 二、Logic Mapping（逻辑映射）

基于 22-26year 档案系统检索，提取金渐成关于 AMZN 的四大核心逻辑链：

### 2.1 "不看 PE 看成长" — 估值哲学

> *"请教下，之前亚马逊200的时候，你说它挺便宜的。能讲讲你是如何评估亚马逊估值吗？从市盈率和回撤幅度来看，它都不算'便宜'。"*
> *"从成长的角度去看，算一下就知道很便宜了。实在不懂，你就看历史百分位。"*
> — 2026-04 月文 L543-545

**核心要义**：
- 金渐成对 AMZN 的估值**不看 P/E 绝对值**，而看**成长性（PEG）和现金流**
- 200 美元时 PEG 极低 → "从成长角度很便宜"
- 当前 $260，PEG 已修正至 1.2–1.4 → 仍在"合理区间"
- 关键判据：OCF 能否持续"滚出"高质量资产（AWS 产能、物流网络、AI 基建）

## 三、Synthesis（综合研判）

### 3.1 "越涨越便宜"检验 — Q1 财报后修正

> *"PEG越低，公司就越便宜。股价明明在涨，估值却越涨越便宜，这就是成长的力量。"*
> — 2024-05-24《拒绝新闻的生活》

**Q1 2026 财报后 PEG 重新测算**：

```
调整后 EPS（剔除 Anthropic 浮盈）≈ $1.24
全年 EPS 上修估计 ≈ $8.50–$9.00
2025A → 2026E EPS 增速 ≈ 22–30%
Forward P/E ≈ 30.7x

PEG = 30.7 / 22 ≈ 1.40（保守）
PEG = 30.7 / 30 ≈ 1.02（乐观）

前版 PEG = 2.8 → 现版 PEG = 1.02–1.40
```

> [!IMPORTANT]
> **结论：PEG 从 2.8 大幅修正至 1.0–1.4 区间。**
> - 1.0 以下 = "越涨越便宜"（经典金渐成标准）
> - 1.0–1.5 = "合理区间"
> - 1.5 以上 = "偏贵"
>
> Q1 超预期财报使 AMZN 从"合理偏贵"回到了**"合理"区间**。作者在 $264 价位明确表示"亚马逊不贵"，与修正后的 PEG 吻合。

## 四、操作框架（2-3-3-2 法适用性评估）

### 4.1 情景 A：已有仓位（负成本 / 接近负成本）

这是金渐成本人当前的状态（进取型账户 4.2–5.5% 占比，接近负成本）。

```
✅ 当前策略：持有 + 等待减仓实现负成本
✅ 减持计划（2-3-3-2 反向操作）：
   - 📌 第一轮：$280+，减仓 10%（5% × 2），实现负成本持股 ← 作者当前计划
   - 📌 第二轮：$300–$320，减仓 15%（5% × 3）
   - 📌 保留：50%+ 底仓作为"负成本永久仓"，长期吃 AWS 利润增长

⚠️ 如出现大幅回调至 $225 以下 → 用减仓释放的资金补仓回来
   （参考作者 2026-03："在225、230减仓的那部分，这轮下跌准备捞点进来补仓"）
```

### 4.2 情景 B：空仓想建仓

```
⚠️ 当前价位 $260 不建议全仓建仓
   - PEG = 1.0–1.4，"合理"但非"便宜"
   - 作者说"亚马逊不贵"，但作者已持有负成本仓位，风险承受力远高于空仓新手

📌 如果非建不可（纪律化 2-3-3-2）：
   阶段        仓位    触发条件              当前距离
   ──────────────────────────────────────────────────
   试探仓      20%    $240 以下回调          ↓ -7.7%
   确认仓      30%    $225 以下              ↓ -13.5%
   主力仓      30%    $210 以下              ↓ -19.2%
   收尾仓      20%    $195 以下              ↓ -25%
   ──────────────────────────────────────────────────

   📌 减仓阈值：如距 $195 仍有 >15%，则第四阶段不执行
```

### 4.3 情景 C：轻仓观察

```
🟡 持有观察，不追高
   - 当前 Q1 财报已消化，短期催化剂释放完毕
   - 下一个关键节点：Q2 2026 财报（约 2026-07/08）
   
   📌 加仓条件：
      ① AWS 增速继续回升（突破 30%）→ 在 $240 以下加仓
      ② 大盘系统性下跌（纳指回调 >10%）→ 在 $220 以下加仓
      
   📌 减仓条件：
      ① AWS 增速再度放缓至 <20% → 考虑减仓 20%
      ② 整体涨至 $300+ → 分批减仓
```
