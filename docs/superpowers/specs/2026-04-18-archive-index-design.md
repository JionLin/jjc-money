# 长期检索索引设计

## 目标

为 `22-25year/` 下的 Markdown 归档建立一套长期可用的索引体系，使后续围绕某个主题、观点、日期或关键词提问时，可以先快速定位相关文章，再回读原文并输出“整理结论 + 关键原文依据”。

## 适用范围

- 归档根目录：`22-25year/`
- 主要文件形态：按月聚合的 Markdown 文件
- 特殊文件：`25-12月.md` 采用不同展示格式，需要按其本地结构单独适配

## 核心原则

1. 回答始终以原文为准，索引只负责加速定位。
2. 索引只保存元数据与基础主题词，不复制正文。
3. 长期查看以 Markdown 为主，优先保证人工可读性与可维护性。
4. 检索流程分层：总索引缩小范围，月度索引定位文章，原文摘录提供依据。

## 索引结构

### 1. 总索引

建议文件位置：`docs/indexes/archive-index.md`

用途：提供整个归档的总览，帮助快速缩小检索范围。

每个月记录以下信息：
- 月份文件名
- 文章数量
- 覆盖日期范围
- 基础主题词汇总
- 指向对应月度索引的链接

### 2. 月度索引

建议文件位置：`docs/indexes/monthly/`

每个月一个 Markdown 索引文件，例如：
- `docs/indexes/monthly/2025-10.md`
- `docs/indexes/monthly/2025-11.md`

每篇文章记录以下信息：
- 日期
- 标题
- 来源文件
- 原文定位标题（优先使用 `## YYYY-MM-DD_金渐成_标题`）
- 基础主题词
- 可选的简短一句话主题概述

### 3. 特殊格式索引

对于 `25-12月.md`，沿用相同目标，但允许采用与该文件实际结构匹配的定位方式，例如发布日期字段、标题字段或章节标题。实现时应先读局部样式，再生成对应索引条目。

## 检索与回答流程

### 步骤 1：查总索引

先根据用户问题中的时间、主题或关键词，从总索引中锁定可能相关的月份。

### 步骤 2：查月度索引

进入候选月份的月度索引，定位可能相关的文章集合。

### 步骤 3：回读原文

打开命中的原始归档文件，只读取相关标题附近的原文段落，不做全量重扫。

### 步骤 4：整理输出

输出采用混合型结构：
- 先给整理后的结论
- 再附关键原文依据

这样既保证答复速度，也保证内容仍然来自文件本身。

## 关键词策略

采用混合策略：
- 先自动提炼基础主题词，写入索引
- 回答具体问题时，再回读正文进行校正

这样可以兼顾长期检索速度与回答准确性，避免只依赖关键词造成偏差。

## 稳定性设计

1. Markdown 作为主索引格式，便于长期查看。
2. 索引不复制正文，减少与原文漂移的风险。
3. 文章定位优先使用稳定标题，而不是近似位置。
4. 对不同历史时期格式不统一的问题，按本地文件样式适配，而不是强行统一。

## 回答能力边界

本设计适合回答以下问题：
- 某个主题过去写过哪些文章
- 某个时期围绕某个点有哪些观点
- 某个问题的核心结论与对应依据是什么

对于更细粒度的问题，例如观点变化、上下文语气差异、跨月演变等，仍需要回读命中文章原文后再整理，不能只靠索引直接作答。

## 后续实施建议

### 第一阶段：建立基础索引

- 生成总索引
- 为每个月生成月度索引
- 对 `25-12月.md` 单独适配

### 第二阶段：形成问答工作流

- 优先查索引
- 再读命中文章原文
- 按“结论 + 原文依据”格式回复

### 第三阶段：增量维护

- 新增归档文件时，只补对应月份索引与总索引摘要
- 已有索引保持不变，避免重复全量重建

## 非目标

- 不在索引中保存完整正文
- 不把索引当成唯一事实来源
- 不为所有历史文件强制统一格式
