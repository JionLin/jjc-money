# 金渐成人物 Skill 设计

## 背景

当前仓库是一个按月份整理的 Markdown 内容归档，核心语料位于 `22-25year/` 与 `26year/`。内容主体围绕金渐成相关文章，同时包含正文与评论区互动。仓库不是应用代码库，也没有构建、测试或运行时环境，因此这项工作本质上是一次基于本地语料的人物蒸馏与 Skill 设计，而不是软件功能开发。

本次任务的目标不是先产出中间调研体系，而是采用“先小样后定稿”的方式，从本地归档中提炼出一个可运行的 `jin-jiancheng-perspective` 人物 Skill。为降低误判风险，需要先形成一份人物内核小样，再据此落成完整 Skill。

## 目标

本设计需要支持：

1. 仅使用 `22-25year/` 与 `26year/` 中的本地语料提炼金渐成的人物画像。
2. 同时把**正文**与**金渐成在评论区中的回复**作为重要一手材料纳入分析。
3. 先产出一份精简“小样摘要”，再据此落成完整 Skill。
4. 最终 Skill 应可直接放入 `.claude/skills/jin-jiancheng-perspective/SKILL.md` 使用。

## 非目标

本次设计不包含以下内容：

- 不做网络搜索，不补充仓库外资料
- 不创建大规模的中间研究目录、外部下载语料或额外调研脚本
- 不把普通读者评论直接当作金渐成观点来源
- 不承诺还原“真实本人”，而是基于当前可见语料提炼可运行的思维与表达框架

## 语料边界与权重

### 语料范围

仅使用以下目录中的 Markdown 文件：

- `22-25year/`
- `26year/`

其中：

- `22-25year/` 中多数文件采用旧结构，常见文章标题形态为 `## YYYY-MM-DD_金渐成_标题`
- `22-25year/25-12月.md` 采用新结构，需要依赖 `# 标题`、`**📅 发布日期**`、`### 💬 评论区` 等局部标记处理
- `26year/` 中新增月度文件也需要按各自实际结构回读，而不是假设统一格式

### 材料分层

为避免把不同性质的内容混在一起，本设计将材料分为三层：

1. **正文**
   - 用于提炼系统性观点、长期主题、重复出现的判断框架
2. **评论区中的作者回复**
   - 作为重要一手材料
   - 用于提炼被追问时的解释路径、应激反应、说服方式、边界感、价值排序
3. **读者评论本身**
   - 仅作为上下文材料
   - 用于理解作者回复针对的问题，但不直接作为金渐成的立场证据

### 权重规则

- 如果某种模式同时出现在正文与作者评论回复中，视为更强证据
- 如果某种模式只出现在评论回复中，优先归类为“决策启发式”或“表达 DNA”，除非它跨多个月份反复出现，才上升为核心心智模型
- 如果某种表述只出现一次，且缺乏跨月份重复，不上升为稳定 worldview

## 总体方案

采用两层式流程：

1. **蒸馏层**
   - 从月度归档中抽取高频主题、判断方式、表达习惯、价值排序、反模式
   - 输出一份“小样摘要”
2. **Skill 组装层**
   - 将小样摘要转为完整人物 Skill，包括触发条件、回答工作流、角色规则、边界说明与风格约束

这样做的核心目的是把“他怎么想”和“Skill 如何运行”分开处理，先确认画像，再固化为可运行文档，避免一次性写完整 Skill 时把错误判断固化进去。

## 数据流设计

### 步骤 1：按月份定位候选内容

优先从月度文件中定位与金渐成相关的文章段落、标题块与评论区块。由于仓库本身就是围绕金渐成展开，实际筛选重点不在“是否相关”，而在“哪些段落能稳定反映其心智模式”。

### 步骤 2：回读正文与评论区回复

读取命中的文章正文与相邻评论区，保留必要上下文，而不是只抽孤立句子。

特别要求：

- 如果引用评论区内容，必须保留“问题 → 回复”的对应关系
- 如需引用一句回复，需连同前后关键语句一起理解，避免把辩解、让步、反问、补充条件切断后误读
- 对 `25-12月.md` 与 `26year/` 中结构差异较大的文件，需要按本地章节标记定位

### 步骤 3：提炼人物内核小样

先产出精简版人物画像，包含：

- 3-5 个心智模型
- 5-8 条决策启发式
- 表达 DNA
- 反模式
- 诚实边界

这一步只输出摘要，不直接写成最终 Skill 文件。

### 步骤 4：组装完整 Skill

在小样获得确认后，再写入完整 `SKILL.md`。完整 Skill 至少包含：

- frontmatter
- 触发条件与使用场景
- 角色扮演规则
- 回答工作流（何时直接回答，何时先回读本地语料）
- 时间排序规则（引用历史材料时默认从近到远）
- 心智模型
- 决策启发式
- 表达 DNA
- 反模式
- 诚实边界
- 来源边界说明

## 提炼规则

### 心智模型

只有满足以下条件的模式，才可作为核心心智模型：

- 跨月份重复出现
- 不只存在于单篇文章
- 能用于解释多个主题下的判断方式
- 不是过于通用、放在谁身上都成立的空泛原则

目标数量控制在 3-5 个，宁少勿滥。

### 决策启发式

用于承载那些更像“常用判断捷径”的模式，例如：

- 面对某类问题时优先关注什么
- 在被质疑时如何澄清
- 遇到分歧时如何划边界

这类内容可以来自正文，也可以来自评论回复；只要它们稳定重复，就可以进入该层。

### 表达 DNA

重点提炼：

- 高频句式
- 常见转折方式
- 解释问题的节奏
- 面对反对意见时的语气
- 是否偏好下定义、举例、反问、拆概念、打标签

评论区回复在这一部分权重较高，因为它更接近即时表达而非成稿修饰。

### 反模式

记录他明确反对、反复批评、或一贯回避的表达和思路。例如某些偷换概念、情绪化判断、短视行为或认知懒惰。反模式有助于让 Skill 不只“像他会说什么”，也知道“他不会怎么说”。

### 诚实边界

最终 Skill 必须明确说明：

- 它只基于当前仓库可见语料，不代表完整人格或全部观点
- 评论区回复能揭示即时判断，但仍受具体问题语境约束
- 单次表达不代表长期稳定立场
- 语料截止到 `26year/` 当前可见文件，不自动包含后续变化

## 输出设计

### 通用排序规则

最终 Skill 在后续答复中，只要涉及引用、整理或比较本地历史材料，默认都按**时间从近到远**组织：先给最新内容，再逐步回溯更早内容。

这个规则同时适用于：

- 正文引用
- 评论区作者回复引用
- 同主题观点演变梳理
- 多篇文章并列比较

只有在用户明确要求按时间正序、追溯起点、或分析最早源头时，才允许改为从远到近。

### 第一阶段交付：小样摘要

在写最终 Skill 前，先给用户确认以下内容：

1. 3-5 个心智模型
2. 5-8 条决策启发式
3. 表达 DNA
4. 反模式
5. 诚实边界

这份摘要是“画像确认层”，用于防止定型过早。

### 第二阶段交付：完整 Skill

用户确认小样后，生成 `.claude/skills/jin-jiancheng-perspective/SKILL.md`，作为最终可运行人物 Skill。

完整 Skill 的行为要求：

- 遇到抽象问题时，直接用提炼出的框架回答
- 遇到明显依赖本地语料的问题时，应先回读仓库内容再作答
- 对无法从现有语料支持的判断，明确说明边界，不伪装成“金渐成一定会这么想”
- 当回答需要引用或归纳本地历史材料时，默认按**时间从近到远**组织答案，让最新表达优先出现，再回溯更早观点
- 如果同一主题存在观点演变，也应按**从近到远**展开，避免较早表述先入为主覆盖较新的判断

## 质量要求

在最终落成 Skill 前，需要满足以下标准：

1. 心智模型来自跨月份重复证据，而不是单次引用
2. 评论区作者回复被明确纳入一手材料，而不是被边缘化
3. 读者评论只作上下文，不越权变成立场证据
4. 正文观点与评论回复的差异被保留下来，不强行抹平
5. Skill 的语气与结构能够体现人物特征，但不拼贴原话、不伪造本人语录
6. 边界说明明确，承认材料有限性

## 风险与应对

### 风险 1：把评论区噪音误当核心观点

应对：只把作者回复纳入核心分析，普通读者评论仅保留上下文作用。

### 风险 2：把偶发表达误当稳定模型

应对：要求跨月份、跨场景重复，单篇或单次表达最多进入启发式层。

### 风险 3：忽略成稿表达与即时回复的差异

应对：正文与评论区回复分层提炼，在最终 Skill 中同时保留“系统表达”与“互动表达”两类证据来源。

### 风险 4：过度拟人化

应对：在诚实边界中明确它是“基于本地语料提炼出的认知代理”，而不是本人替身。

## 实施范围判断

本设计聚焦单一交付物：`jin-jiancheng-perspective` 人物 Skill。它不需要拆成多个独立子项目，规模适合在一个 spec → 小样 → 最终 Skill 的链路内完成。
