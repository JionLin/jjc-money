# jjc-money 智能数字分身 (v1 架构设计)

**文档状态**: 草案 (Draft) / 探索期 (Explore Phase)  
**目标**: 构建一个基于 `jinjian-perspective` 的 AI 对话应用，前端提供交互页面，后端(Java)提供逻辑引擎与大模型接入。

## 2. 前端选型与部署 (Frontend & Deployment)

### 2.1 技术栈
- **核心框架**: Vue 3 (搭配 Vite 快速构建)。轻量且非常适合开发 SPA (单页应用) 聊天界面。
- **UI 交互**: 类似 ChatGPT 的简洁对话框，后续迭代支持表单和上传。
- **渲染**: 支持 Markdown 和代码高亮渲染。

### 2.2 部署方案 (Single Jar Deployment)
- **前后端耦合部署**: 前端项目执行 `npm run build` 后，将其 `dist` 产物拷贝至 Spring Boot 的 `src/main/resources/static/` 目录下。
- **优势**: 无需部署独立的 Nginx 或 Node 服务器，完全消除跨域问题 (CORS)。最后只需运行一个 Java `.jar` 包即可提供全部服务，极其适合个人工具的低成本维护。

## 3. 核心机制：Prompt 热更新 (Hot Reload)

为了最大化 Prompt Engineering 的调试效率，后端系统必须支持 **热更新**：

- **读取时机**: Java 后端不能在启动时缓存 `SKILL.md`。必须在每次接收到用户提问时，动态从磁盘 (相对路径) 读取 `.agent/skills/jinjian-perspective/SKILL.md` 的最新内容。
- **收益**: 调整“金渐成”的语气、修改 2-3-3-2 规则或更新锚点数据后，只需要保存文件，在网页上点“重新生成”，大模型即可立刻应用新规则，无需重启后端服务。

---

## 4. 历史语料接入方案：Function Calling 工具调用

抛弃沉重的全量向量数据库 (RAG) 方案，采用轻量级的 **大模型工具调用 (Function Calling)** 机制：

- **后端实现**: 在 Spring AI 中注册一个 Java 方法 (例如 `searchLocalArchive(@ToolParam String keyword)`)，该方法在本地文件系统 (`22-25year/`) 中执行简单的文本检索并返回。
- **智能交互**: 在 Prompt 中赋予大模型这个工具的使用权。如果用户提问属于历史观点（如“以前是怎么看AAPL的”），大模型自主决定调用该工具抓取历史文章，然后结合 `SKILL.md` 的规范生成“金渐成”视角的最终回答。
- **优势**: 保持原文语境不被打散，极大地减轻了工程复杂度。
